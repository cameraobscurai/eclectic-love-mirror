# PR 3 ‚Äî Share-link hardening

Taking "yes" as go-ahead for PR 3 (higher-priority of the two remaining). PR 2 (email cron safety-net) stays queued.

## Goal

Move `style_boards` share tokens from "raw UUID stored in a queryable column" to "high-entropy secret, hashed at rest, with expiry + revocation + a light rate limit." Guessing 122-bit UUIDs is already infeasible, so this is defense-in-depth, not an active-exploit fix ‚Äî but a leaked link is currently forever.

## Scope

**In**
1. Token generation: 256-bit random, base64url encoded (`crypto.getRandomValues` in generator).
2. Storage: add `share_token_hash` (sha-256 hex), `share_token_expires_at`, `share_token_revoked_at`. Backfill existing rows by hashing their current token so live links keep working. Keep raw `share_token` column temporarily for backfill window, then drop in a follow-up migration once no active links reference it. (Not dropped in this PR ‚Äî safer.)
3. Lookup: `getStyleBoardByToken` hashes the incoming token and looks up by `share_token_hash`, rejects if `revoked_at IS NOT NULL` or `expires_at < now()`.
4. Issue path: `generateShareToken` (in `src/lib/studio.functions.ts`) returns the raw token to the admin ONCE, stores only the hash + expiry (default 90 days, configurable per call).
5. Revoke path: new admin server fn `revokeShareToken({ boardId })` sets `share_token_revoked_at = now()`. Wire a "Revoke link" button into the existing admin board UI.
6. Rate limit on `/stylebrief/$token`: light IP-based counter in a new `share_token_access_log` table (or reuse existing if present) ‚Äî 30 requests / 5 min / IP, 429 otherwise. Only counts failed lookups; successful lookups don't burn budget.
7. Response headers on `/stylebrief/$token`: `Referrer-Policy: no-referrer`, `X-Robots-Tag: noindex` (route already sets meta noindex; add header too).

**Out (deferred)**
- Dropping the raw `share_token` column ‚Äî separate migration after 1 release cycle.
- Per-recipient tokens / one-time links ‚Äî not asked for.
- Full audit log UI ‚Äî table exists, admin surface later.

## Files touched

## Technical notes

- Hash = `sha256(token)` hex, computed with Web Crypto (`crypto.subtle.digest`) in the server fn. No pepper ‚Äî token itself is 256-bit, so hashing is only there to neutralize DB-dump leaks.
- Backfill in migration: `UPDATE style_boards SET share_token_hash = encode(digest(share_token, 'sha256'), 'hex'), share_token_expires_at = now() + interval '90 days' WHERE share_token IS NOT NULL AND share_token_hash IS NULL;` (requires `pgcrypto`, already available).
- Rate-limit table: `(ip inet, window_start timestamptz, count int)` with an index on `(ip, window_start)`. Cleaned by a simple `DELETE WHERE window_start < now() - interval '1 hour'` in the same server fn on write.
- No client-side changes to the public board page beyond header + error state ‚Äî DOM stays intact.
